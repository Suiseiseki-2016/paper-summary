# SWE-Search Methodology 总结

## 核心框架概述

SWE-Search 是一个**多代理系统**，通过整合**蒙特卡洛树搜索(MCTS)**、**价值估计**和**审议决策**来解决复杂的软件工程任务。该系统旨在模拟人类软件工程师的迭代工作流程。

---

## 四大核心组件

### 1. SWE-Search 框架与行动代理 (Action Agent)

| 特性 | 描述 |
|------|------|
| **基础框架** | 基于 moatless-tools 框架，具有类 git 的提交树结构 |
| **状态空间** | 灵活的状态空间，支持高效回溯到先前状态 |
| **动作层次** | 两层结构：动作类型 (Search/Plan/Edit) + 具体动作 |
| **关键改进** | Plan 状态可灵活转换到任意其他状态，而非仅限于 Edit |
| **测试能力** | 代理可自主执行和创建测试，测试结果纳入价值函数 |

### 2. 价值函数代理 (Value Agent)

```
输出 = (v_t, ε_t) = V(s_t, a_t, {s_i}, {a_i})
```

| 功能 | 说明 |
|------|------|
| **数值估计** | 输出状态 - 动作对的效用值 v_t (-100 到 100) |
| **定性反馈** | 生成自然语言解释 ε_t，用于后续改进 |
| **事后反馈** | 解释可作为 hindsight feedback 指导父节点重新扩展 |
| **轨迹感知** | 考虑完整的历史状态和动作序列进行评估 |

### 3. 搜索算法 (Modified MCTS)

**改进的 UCT 选择准则：**

$$UCT(s,a) = V(s,a) + C\sqrt{\frac{\ln N(s)}{N(s,a)}} + \alpha e^{-\beta(d-1)} - \gamma\sqrt{d}$$

| 组件 | 作用 |
|------|------|
| **V(s,a)** | 状态 - 动作对的价值估计（利用） |
| **探索项** | 平衡已知高奖励动作与较少访问状态 |
| **早期深度奖励** | αe^(-β(d-1)) 鼓励浅层广泛探索 |
| **后期深度惩罚** | -γ√d 防止过度深入搜索 |

**搜索策略模拟工程师行为：**
- **早期探索**：广泛探索多种方法
- **收敛利用**：聚焦最有效的解决路径
- **快速放弃**：启发式规则放弃连续低奖励节点

### 4. 判别器代理 (Discriminator Agent)

| 特性 | 配置 |
|------|------|
| **输入** | 最多 5 个搜索生成的候选解决方案 |
| **机制** | 多代理辩论，各代理为不同方案辩护 |
| **辩论轮数** | 3 轮辩论 + 结论阶段 |
| **代理数量** | 5 个判别器代理 |
| **效果** | 将正确方案选择准确率从 73% 提升至 84% |

## 问题形式化

任务定义为元组 **M = (S, C, A, V, P, p₀, ρ)**：

- **S**: 状态空间（文件上下文、代码库状态）
- **C**: 上下文空间（仓库元数据、问题描述）
- **A**: 动作空间（两层层次结构）
- **V**: 价值函数（指导决策）
- **P**: 状态转移函数
- **目标**: 最大化轨迹累积奖励 **max_π E[ΣR(s_t, a_t, c)]**

---

## 关键创新点

1. **混合价值函数**：LLM 同时输出数值估计和自然语言解释
2. **灵活状态转换**：打破严格的状态机限制，支持非线性工作流
3. **事后反馈机制**：价值解释用于迭代改进搜索过程
4. **多代理辩论**：通过结构化辩论提高最终方案选择质量
5. **推理时计算扩展**：通过增加搜索深度提升性能，无需更大模型

## 性能提升

在 SWE-bench Lite 基准测试中，SWE-Search 相比基线代理实现 **23% 相对性能提升**，且开源模型（如 Qwen-2.5-72B）可接近 GPT-4o 的性能水平。
