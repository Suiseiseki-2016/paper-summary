### 1. 核心思想：自适应检索增强证明 (Adaptive RAP)
Rango 的核心理念是在证明生成的**每一步**都进行检索增强。与以往仅在证明开始时检索前提（如引理）的方法不同，Rango 会根据当前的**证明状态 (Proof State)**，动态地从当前项目中检索相关的**已完成证明 (Proofs)** 和 **引理 (Lemmas)** 作为上下文输入给大语言模型 (LLM)。这使得模型能够适应特定项目的证明风格以及证明过程中的演变状态。

### 2. 系统架构
Rango 由两个主要子系统组成：
*   **策略生成器 (Tactic Generator)**：负责生成单个证明步骤（即 Coq 中的 tactic）。
*   **搜索器 (Searcher)**：利用策略生成器组合生成的策略，搜索完整的证明路径。

### 3. 策略生成器 (Tactic Generator) 组件
策略生成器包含三个关键组件，用于在每一步生成下一个 tactic：

*   **A. 证明检索器 (Proof Retriever)**
    *   **功能**：从当前项目的“证明库 (Proof Bank)"中选择相关的已完成证明作为上下文。证明库包含当前文件及其依赖文件中之前的证明。
    *   **算法**：使用 **BM-25** 信息检索技术。
    *   **机制**：将当前的“证明状态”视为查询 (Query)，将证明库中证明的中间状态视为文档 (Documents)。通过计算标识符 (identifiers) 的词频相似度，选出 $k$ 个最相关的证明状态所在的证明。
    *   **目的**：为 LLM 提供类似证明结构的灵感。

*   **B. 引理检索器 (Lemma Retriever)**
    *   **功能**：从当前项目的“引理库 (Lemma Bank)"中选择可能直接使用的引理陈述（不包含证明过程）。
    *   **算法**：使用 **TF-IDF** 稀疏检索算法。
    *   **机制**：将当前证明状态作为查询，引理库中的引理作为文档，计算相关性得分，选出 $j$ 个最相关的引理。
    *   **目的**：为 LLM 提供可用的辅助前提。

*   **C. 语言模型 (Language Model)**
    *   **模型**：基于 **DeepSeek-Coder 1.3B** (decoder-only) 进行微调。
    *   **输入**：检索到的相关证明、检索到的相关引理、定理陈述、当前的证明脚本（已完成的步骤）以及当前的证明状态。
    *   **训练**：使用 CoqStoq 数据集构建微调样本。每个样本包含上述输入作为 Prompt，下一个 tactic 作为 Target。损失函数仅计算在 Target 上。
    *   **Token 限制**：对输入的各部分（证明、引理、脚本、状态）和输出设置最大 Token 数，超出部分进行截断（如保留证明状态的最长后缀）。

### 4. 搜索器 (Searcher)
*   **算法**：使用 **Rollout Search (展开搜索)**。
*   **流程**：
    1.  从策略生成器中采样一个 tactic（使用 temperature sampling）。
    2.  将该 tactic 添加到当前证明中，并使用 Coq 检查。
    3.  **状态判断**：
        *   **Complete (完成)**：Coq 显示无剩余目标，搜索成功。
        *   **Invalid (无效)**： tactic 导致错误，开始新的 rollout（重置搜索路径）。
        *   **Incomplete (未完成)**：继续当前 rollout，采样下一个 tactic。
    4.  重复直到找到完整证明或超时（设置为 10 分钟）。
*   **对比**：论文评估发现 Rollout Search 优于 Best-First Search（如 Beam Search）。

### 5. 数据集构建：CoqStoq
为了训练和评估 Rango，作者构建了一个新的大规模 Coq 数据集 **CoqStoq**。
*   **来源**：从 GitHub 挖掘了 2,226 个开源 Coq 项目（截至 2023 年 11 月 5 日）。
*   **验证**：使用 CoqPyt 工具验证文件是否能在 **Coq 8.18** 版本下成功编译。只保留无编译错误的文件。
*   **提取**：提取每个证明步骤的文本表示、对应的证明状态及上下文（使用的前提）。
*   **划分**：
    *   **训练集**：包含大部分仓库和数据，用于微调模型。
    *   **Benchmark (测试集)**：包含 CoqGym 中兼容 Coq 8.18 的项目、CompCert 以及 Coq-Community 中长期维护的项目。
    *   **验证集**：用于调整超参数。
*   **防泄漏**：如果训练集中的定理陈述与验证集/测试集完全匹配，则从训练集中剔除该文件，防止数据泄漏。
*   **规模**：共包含 196,929 个定理和 2,225,515 个证明步骤。

### 总结
Rango 的方法论创新在于将**检索增强 (RAG)** 深度集成到证明生成的**每一步**中，不仅检索前提（引理），还检索**相似的证明过程**。通过 BM-25 和 TF-IDF 分别检索证明状态和引理，结合微调的 LLM 和 Rollout 搜索策略，实现了在 CoqStoq 基准测试上优于现有工具（如 Tactician, Proverbot9001）的性能。
