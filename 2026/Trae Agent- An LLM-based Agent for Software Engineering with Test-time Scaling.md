该论文提出 Trae Agent，首个面向仓库级软件问题修复的基于智能体的集成推理（ensemble reasoning）框架。其核心思想是将集成推理形式化为一个最优解搜索问题，并通过模块化智能体架构应对两大关键挑战：大规模候选补丁空间的高效探索，以及对整个代码仓库的全局理解。

首先，Trae Agent 将问题建模为三阶段流程：补丁生成、补丁剪枝与补丁选择。在补丁生成阶段，系统采用一个专用的 Coder 智能体，在给定 GitHub 问题描述和代码库后，通过高温度采样或混合多个大语言模型（如 Gemini 2.5 Pro、Claude 3.7 Sonnet、GPT-4.1）并行生成多样化的候选补丁。该智能体配备文件编辑、Bash 终端、顺序推理等工具，能在 Docker 环境中完整复现问题、定位根因、生成修复并验证，从而产出高质量、多样化的候选集。

其次，为应对候选集规模增大带来的搜索难度，Trae Agent 引入分层剪枝机制。该机制包含两个子策略：一是基于补丁归一化的去重（patch deduplication），利用 unidiff 解析原始补丁，去除空格、注释等无关差异，合并语义等价的候选；二是基于回归测试的过滤（regression testing），由 Tester 智能体从原始代码库中提取通过的测试用例作为回归测试集，并对每个候选补丁执行这些测试，剔除破坏现有功能的错误补丁。实验表明，该剪枝策略平均可将候选集规模减少近 30%，同时显著提升后续选择的准确率。

最后，在补丁选择阶段，Trae Agent 设计了一个 Selector 智能体，模拟真实开发者对代码的理解过程。该智能体通过工具访问完整代码库，主动检索问题描述中提及的文件、补丁修改的代码段以及相关依赖模块，构建静态上下文理解；同时可生成并执行单元测试，收集运行时反馈以建立动态行为理解。在此基础上，系统采用多数投票（majority voting）策略：并行运行 Selector 智能体多次，每次独立选择一个补丁，最终选取得票最高的作为最终输出。该机制有效缓解了大语言模型的幻觉问题，增强了选择的稳定性。

作者在 SWE-bench Verified 基准上对 Trae Agent 进行了全面评估。结果表明，其 Pass@1 得分为 75.20%，在官方排行榜上位列第一。与四种先进集成方法（Augment、DeiBase 及其剪枝增强版）相比，Trae Agent 在所有设置下均取得显著且稳定的性能优势，平均提升 10.22%。消融实验证实，剪枝组件（含去重与回归测试）和基于智能体的选择机制（含多数投票）均对整体性能有显著贡献。此外，随着集成规模增大，Trae Agent 的性能持续提升，而基线方法则出现性能下降，说明其架构具备良好的可扩展性。

综上，Trae Agent 通过模块化智能体协同工作，实现了对大规模候选空间的有效压缩与对仓库级上下文的深度理解，为复杂软件工程任务中的集成推理提供了一个高效、鲁棒且可扩展的新范式。项目已开源，获得社区广泛关注。
