该论文提出 **MapCoder**，一种基于**多智能体协同**的代码生成框架，专门用于解决**竞赛级编程问题**。其核心思想是**模拟人类程序员的完整问题解决周期**，通过四个专门设计的 LLM 智能体（Agent）分工协作，分别负责**回忆相关案例、制定计划、生成代码和调试修复**，从而系统性地提升复杂代码生成任务的成功率。

MapCoder 从一个问题描述开始，首先由**检索智能体**（Retrieval Agent）自主生成若干个与当前问题相似的、带有完整解决方案（包括代码和计划）的示例，而非依赖外部数据库或人工标注数据。接着，**规划智能体**（Planning Agent）利用这些示例和问题描述，为当前问题生成一个或多个详细的解决步骤（Plan），并为每个计划附上一个置信度分数。

随后，**编码智能体**（Coding Agent）选取置信度最高的计划，并将其翻译成可执行代码。该代码会立即在问题提供的**样例输入/输出**（sample I/O）上进行测试。如果测试通过，则输出为最终答案；如果失败，则交由**调试智能体**（Debugging Agent）进行修复。调试智能体在修复过程中，会同时参考原始计划和失败的测试报告，进行有针对性的修正，确保修复后的代码既符合逻辑又通过测试。

整个流程采用**动态遍历机制**：如果在预设的调试次数内未能修复代码，系统会回退到规划智能体，尝试下一个置信度最高的备用计划。这一过程循环往复，直到找到正确解或耗尽所有尝试机会。

该方法有效解决了现有代码生成方法的两大局限。首先，通过**自主检索和多计划生成**，为模型提供了高质量的上下文学习信号，克服了零样本提示信息不足的问题。其次，调试过程**仅依赖问题自带的样例 I/O**，避免了像 Reflexion 等方法那样需要额外生成测试用例，从而规避了因生成错误测试用例而导致代码被错误修正的风险。

作者在八个具有挑战性的代码生成基准上（包括 HumanEval、MBPP 等基础任务，以及 APPS、CodeContests、xCodeEval 等竞赛级任务）对 MapCoder 进行了评估。结果显示，MapCoder 在所有基准上均**显著超越现有最强基线**，并取得了**新的技术纪录**（例如，在 HumanEval 上达到 93.9% Pass@1，在 CodeContests 上达到 28.5%）。消融实验证明，四个智能体缺一不可，其中**调试智能体**和**规划智能体**的贡献最为关键。此外，MapCoder 在不同编程语言和不同难度级别的问题上均表现出色，且能有效适配 ChatGPT、GPT-4、Gemini Pro 和开源模型 Mistral 等多种 LLM，展现出强大的通用性和鲁棒性。

与需要复杂工程或离线训练的方案不同，MapCoder 完全基于**提示工程**（prompting）实现，无需任何模型微调，提供了一条高效、通用且可复现的路径，以构建能解决复杂编程问题的智能系统。项目代码已开源。
