### 1. 核心背景与问题
*   **数据墙（Data Wall）：** 在 CUDA 等成熟平台上，LLM 表现优异，因为有大量训练数据。但在 NPU（如 Ascend C）等新兴领域特定架构（DSA）上，公开代码少、文档晦涩，导致 LLM 性能断崖式下跌（例如 GPT-5.2 在 CUDA L1 任务上准确率 92%，在 Ascend C 上降至 14%）。
*   **冷启动挑战：** 传统的监督微调（SFT）需要大量专家数据，成本高昂；基于参数的强化学习样本复杂度高；传统的检索增强生成（RAG）在数据库稀疏时效果不佳。
*   **目标：** 如何在没有专家演示和昂贵微调的情况下，让通用模型自主掌握数据稀缺的内核合成任务。

### 2. 解决方案：EvoKernel 框架
EvoKernel 将内核合成建模为基于记忆的强化学习任务，通过**价值驱动的检索机制**来自动化从初始草稿到持续优化的生命周期。

*   **两阶段流程：**
    1.  **冷启动草稿（Cold-Start Drafting）：** 目标是获得功能正确的初始内核。
    2.  **持续优化（Continual Refining）：** 目标是降低延迟，提升性能。
*   **基于记忆的 MDP（M-MDP）：** 代理通过检索历史经验来指导 LLM 生成代码，而不是更新模型权重。
*   **价值驱动检索（Value-Driven Retrieval）：**
    *   不同于传统的基于语义相似度的检索，EvoKernel 学习阶段特定的 **Q 值**。
    *   $Q_1$（草稿阶段）：评估记忆项对功能正确性的贡献。
    *   $Q_2$（优化阶段）：评估记忆项对延迟优化的贡献。
    *   使用统一的蒙特卡洛更新规则，根据验证器反馈动态调整 Q 值。
*   **多门控验证器（Multi-Gate Verifier）：** 提供细粒度反馈，包括防黑客检查（防止绕过 NPU 计算）、编译检查、功能正确性检查和延迟测量。

### 3. 关键技术创新
1.  **统一的草稿 - 优化流水线：** 在共享记忆库上运行，从可行性驱动过渡到延迟驱动。
2.  **进化价值驱动检索：** 引入 Q 值量化记忆效用，使代理能根据当前目标（是求对还是求快）动态检索最有用的经验。
3.  **跨任务记忆共享：** 允许从简单任务（L1）向复杂任务（L2）迁移知识，实现“涌现式课程学习”。

### 4. 实验结果
论文在 Ascend C 变体的 KernelBench 上进行了评估，对比了 GPT-5.2、DeepSeek-V3.2 和 Qwen3-Coder-30B 等模型。

*   **正确率提升：**  frontier 模型（GPT-5.2）的正确率从 **11.0% 提升至 83.0%**，编译率从 11.0% 提升至 98.5%。
*   **性能优化：** 通过迭代优化，中位加速比达到 **3.60 倍**（相对于初始可行草稿）。
*   **泛化能力：**
    *   **跨难度迁移：** 先在 L1 任务上积累记忆，再处理 L2 任务，比直接从 L2 开始学习快得多（正确率 64% vs 34%）。
    *   **跨模型迁移：** 用 GPT-5.2 构建的记忆库可以显著提升弱模型（如 DeepSeek、Qwen）的表现。
    *   **分布外泛化：** 在 Attention Set 和 DeepSeek mHC 内核等未见过的任务上依然有效。
*   **对比基线：** 显著优于 Pass@k、迭代优化（Refinement）以及拥有文件/Shell 访问权的 Codex 代理。

### 5. 结论与意义
*   **无需微调：** 证明了通过价值引导的经验积累，通用模型可以在不更新权重的情况下掌握 niche 硬件生态。
*   **民主化专家知识：** 该方法有助于缓解数据稀缺领域的专家短缺问题，使 AI 系统能以最少数据适应新领域。
*   **未来方向：** 可扩展至其他 DSL，探索知识蒸馏以减少对大模型的依赖，以及引入更密集的奖励信号。

**一句话总结：** EvoKernel 通过引入基于强化学习 Q 值的动态记忆检索机制，成功解决了 LLM 在数据稀缺的 NPU 内核合成任务中的冷启动难题，大幅提升了代码正确率和执行效率。
